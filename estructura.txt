ESTRUCTURA DEL PROYECTO — curp-validator
=========================================

Objetivo: registrar ciudadanos y validar que el PDF de CURP subido
corresponda a la persona registrada, usando PDFBox + Tesseract OCR.

Raíz del repositorio: curp-validator/
Remoto: https://github.com/2201300x-create/curp-validator.git

MODELO DE DATOS
---------------

Tabla: ciudadanos
  - id (PK)
  - nombre, primer_apellido, segundo_apellido
  - curp (única, 18 caracteres, regex validada)
  - documento_validado (boolean, default false)
  - fecha_registro (auto al crear)

Tabla: documentos_rechazados
  - id (PK)
  - ciudadano_id (FK → ciudadanos)
  - ruta_documento
  - motivo_rechazo
  - curp_detectada (opcional)
  - fecha_rechazo (auto al crear)


FLUJO DE VALIDACIÓN
-------------------

1. Usuario registra ciudadano (nombre, apellidos, CURP). Máximo 10 en fase de prueba.
2. Usuario sube PDF del documento oficial de CURP.
3. PdfService extrae texto embebido; si no hay CURP, OcrService aplica Tesseract.
4. DocumentoCurpAnalyzer verifica que sea constancia oficial (RENAPO, CONSTANCIA, etc.).
5. Se extrae la CURP y se compara con la registrada.
6. Se verifica que nombre y apellidos coincidan con el documento.
7. Si coincide todo → documento_validado = true.
8. Si no → PDF en evidencias/ + registro en documentos_rechazados.
